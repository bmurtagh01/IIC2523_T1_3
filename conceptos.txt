Conceptos:

Den un contexto de para qué sirven estas herramientas, sus usos en la actualidad, número de usuarios,
o cualquier otra información que ayude al lector a entender cuál es la utilidad de estos.
De respuestas concisas, no más de 6 lineas.

-Amazon Web Service: Es un servicio de nube de Amazon el cuál ofrece más de 200 servicios como potencia de computo, almacenamiento de bases de datos o servicios de inteligencia artificial.
Hoy en día miles de empresas, pymes a multinacionales, usan los servicios de AWS para albergar sus servicios TI dada la cantidad de funcionalidades que tiene y su capacidad de escalabilidad, 
flexibilidad, optimización y medición. Levantar servicios TI como un servidor tomaba de semanas a meses según la escala, hoy grandes o pequeños proyectos pueden ser rápidamente levantados
en tan solo horasy a reducidos costos gracias a servicios en la nube como AWS.

-AWS IAM Roles: IAM refiere a Identity & Access Management nos permite manejar roles dentro de la plataforma de AWS al poder delegar a otros usuarios niveles de acceso personalizado según su
función. Esto permite el desarollo de una organización sea segura ya que no queremos que un nuevo interno sea capaz de borrar todos los servicios de AWS ya sea por error o mala intención.
Tambien nos permite dar acceso temporal a recursos de la organización y así no tener que compartir credenciales importantes.

-AWS EC2: Es un servicio que provee capacidad de procesamiento computacional dimensionable en la nube, permitiendo escalar de manera rápida la capacidad, como que el requerimiento de 
procesamiento de computo cambie, solo basta provisionar los sistemas con los bloques necesarios para escalar y pagar por la capacidad que se utiliza. AWS provee de todos estos servicios 
para lograr escalar de manera sencilla y sin un alto costo.

-AWS S3:es un servicio de almacenamiento de objetos creado para almacenar y recuperar cualquier volumen de datos 
desde cualquier ubicación. Es un servicio de almacenamiento sencillo que ofrece durabilidad, 
disponibilidad, rendimiento, seguridad y escalabilidad prácticamente ilimitada a costos muy reducidos. Se puede 
usar para almacenar cualquier el tipo y la cantidad de datos, leer el mismo fragmento de datos un millón de veces 
o solo en caso de recuperación de desastres de emergencia.

-AWS RDS: un servicio administrado de base de datos relacional que puede utilizar el código, las aplicaciones 
y las herramientas que utiliza actualmente con las bases de datos existentes. Amazon RDS se encarga de las 
tareas habituales de las bases de datos, como el aprovisionamiento, las revisiones, las copias de seguridad, 
la recuperación, la detección de errores y la reparación.

-AWS Lambda: es un servicio informático que permite ejecutar código sin aprovisionar ni administrar servidores.
realiza todas las tareas de administración de los recursos de computación, incluido el mantenimiento del servidor
 y del sistema operativo, el aprovisionamiento de capacidad y el escalado automático, así como las funciones de
  registro.

-CircleCI: Plataforma de Continuous Integration and Development (CI/CD). Con esto es posible realizar ciertas revisiones del código realizado de forma automática y frecuente.
Fomenta el integrar el código realizado a un repositorio principal de forma frecuente y así ejecutar autómaticamente acciones sobre este,
y por otro lado realiza el proceso de deploy de una forma automatizada, acelerando la distribución de resultados al producto final.
Es preferida por sobre 1 millón de usuarios, siendo la herramienta más popular de este tipo.

-Jenkins: Servidor de automatización autocontenido para proyectos, open source y desarrollado en Java. Se utiliza para automatizar tareas
relacionadas con el desarrollo de software (desarrollo, testing, delivery y deploy).
Permite crear una pipeline de CI/CD personalizada mediante el uso de plugins para adaptarse a cada proyecto.

-Kubernetes: Plataforma de sistema distribuido para automatizar el proceso de deployment, manejo y ajuste de escala de aplicaciones
basadas en contenedores. Con esta plataforma, es posible agrupar y administrar servidores que ejecutan contenedores de Linux de
forma fácil y eficiente. Desarrollada inicialmente por Google, actualmente es open source.

-Kubernetes Pods: Unidad básica de cómputo con la que se trabaja en Kubernetes. Una pod (o cápsula) es un grupo de uno o más contenedores
que poseen recursos compartidos, y se encuentran en el mismo equipo, y hace sentido que se ejecuten de forma sincronizada debido a
su alto acoplamiento. El objetivo de tener Pods en Kubernetes es acoplar contenedores que deben trabajar de forma sincronizada.

-DNS: Sigla para "Sistema de Nombres de Dominio". Es un sistema de nomenclatura descentralizado para dispositivos conectados a redes IP.
Su objetivo es enlazar los identificadores de un equipo (Dirección IP) que poseen valores complejos de recordar para una persona,
con un nombre de Dominio legible para humanos, de forma de poder acceder al recurso mediante su nombre de dominio.
En palabras más simples, es como el "phonebook" o "guía telefónica" del Internet.

-NIC Chile: Centro encargado de registrar y administrar los nombres de dominio que terminan en ".cl", extensión que identifica a las
páginas de Chile en Internet. Pertenece a la Facultad de Ciencias Físicas y Matemáticas de la Universidad de Chile,
y posee a la fecha alrededor de 724000 nombres registrados con el dominio CL.
